FAQ:
What is this application for?
This application allows for the translation between an SBML model and its equivalent BioNetGen definition.

What kind of files should I upload?
You should upload an SBML definition file, which is an XML file that follows the SBML convention. Uploading any other file will result in an error.

What is the atomize option for?
Leaving the atomize option unmarked allows for a raw translation of the SBML file. The resultant file will not have any component information. When atomization is enabled the application will attempt to extract as much context information from the original SBML file as possible to obtain a richer BNGL definition.

What does the reaction definition files mean?
The reaction definition files contain an specification on the structure and general guidelines that the translator will use in order to understand the SBML file. When unsure, select reactionDefiniton7.json. Bellow is a definition of the kind of information each file contains.

Isn't there an easier way of doing this?
In the near future we will enable the translator to automatically detect the patterns it is supposed to use. For the time being, the easiest is to first attempt a non-atomized translation and check the resulting BNGL file. From here one would be able to see what kind of naming patterns appear in the file, and select a definition file accordingly. Alternatively, reactionDefinition7 has been found to statistically work with a large percentage of the models in BioModels.
